# Корпоративный кластер GraphRAG «Научный клубок» (First Finch)

Проект представляет собой распределенную поисково-аналитическую GraphRAG систему для R&D-сектора «Норникеля» и НИИ «Гипроникель». 
Система связывает разрозненные научно-технические статьи, регламенты, отчеты об экспериментах, данные об оборудовании и контакты экспертов в единую базу знаний.

## 🖥️ Архитектура распределенного кластера

Вся логика разделена между тремя физическими машинами (ноутбуками), объединенными в Mesh-сеть через **Tailscale**:

1. **Ноут 1: Acer N200 (100.126.216.18 / 100.64.0.20) — Графовая база данных Neo4j**
   - Роль: Хранение строгой онтологии знаний, числовых параметров экспериментов и связей между сущностями.
   - Запуск: Neo4j в Docker-контейнере с жестким ограничением RAM.

2. **Ноут 2: Acer N150 (100.70.36.78 / 100.64.0.30) — Векторный индекс и Воркер парсинга**
   - Роль: Полнотекстовый и семантический векторный поиск, хранение чанков документов и проведение фонового парсинга.
   - Запуск: ChromaDB Server в Docker-контейнере или локально.

3. **Ноут 3: ASUS Master (100.71.14.9 / 100.64.0.10) — Головной сервер и Веб-интерфейс**
   - Роль: Бэкенд FastAPI (оркестрация RAG, интеграция с Yandex AI Studio) + Фронтенд Streamlit с интерактивной визуализацией графов.

## 🚀 Инструкция по запуску

### 1. Настройка Ноута 1 (Графовая база - Acer N200)
Убедитесь, что Tailscale запущен, и выполните:
```bash
docker compose -f docker/docker-compose.neo4j.yml --env-file .env up -d
```
Neo4j будет доступна по Bolt-протоколу: `bolt://<NEO4J_HOST>:7687`

### 2. Настройка Ноута 2 (Векторная база - Acer N150)
Убедитесь, что Tailscale запущен, и выполните:
```bash
docker compose -f docker/docker-compose.chroma.yml --env-file .env up -d
```
ChromaDB будет доступна по адресу: `http://<CHROMA_HOST>:8000`

### 3. Настройка Ноута 3 (Мастер - ASUS)
1. Склонируйте репозиторий и перейдите в директорию:
   ```bash
   git clone <url_репозитория>
   cd first-finch-nornickel
   ```
2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   # Для Windows:
   venv\Scripts\activate
   # Для Linux/Mac:
   source venv/bin/activate
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Настройте файл `.env`, вставив актуальные IP-адреса Tailscale и ваш API-ключ Yandex AI.
5. Запустите тест сетевого подключения и API:
   ```bash
   python scripts/test_cluster.py
   ```
6. Запустите бэкенд FastAPI и UI Streamlit:
   ```bash
   # Запуск бэкенда
   uvicorn app.main:app --host 0.0.0.0 --port 8000
   
   # В новом терминале запуск UI
   streamlit run app/ui/streamlit_app.py --server.address 0.0.0.0 --server.port 8501
   ```

## 📊 4 эталонных вопроса для проверки системы

1. **Обессоливание воды:** Подбор методов под строгие числовые параметры (сульфаты/хлориды/Ca/Mg/Na 200–300 мг/л, сухой остаток ≤1000 мг/дм³).
2. **Циркуляция католита:** Технические решения при электроэкстракции никеля и оптимальная скорость потока.
3. **Драгметаллы в штейне:** Распределение Au, Ag и МПГ между медным/никелевым штейном и шлаком за последние 5 лет (с 2021 года).
4. **Закачка шахтных вод:** Способы закачки в глубокие горизонты (сравнение РФ vs Мировая практика) и их ТЭП (Технико-экономические показатели).
